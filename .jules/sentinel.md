## 2024-05-14 - Remove Remote Code Execution and Path Traversal Vulnerabilities
**Vulnerability:** A critical Remote Code Execution (RCE) vulnerability existed in `js/engine/script-manager.js` where the `source` property of an inline script was executed via `new Function()`. Furthermore, the `type` property in script components lacked validation, allowing potential Path Traversal attacks when dynamically importing modules.
**Learning:** Executing code directly from JSON configuration files (especially unvalidated or from untrusted sources) inherently creates an RCE vulnerability. Path Traversal attacks occur when untrusted input is used directly in file paths or imports. Both violate the principle of "trust nothing while verifying everything."
**Prevention:** Remove features that rely on dynamic code execution (like `new Function()`, `eval()`). Always load scripts from predefined, validated external files. Use strict regex validation (e.g., `/^[a-zA-Z0-9_-]+$/`) for any user-provided or configurable input used in file paths or module imports to ensure only safe characters are permitted.
